# QuickTab Spotlight 风格 UI 优化方案

## 目标

将 QuickTab 主界面调整为高度贴近 macOS Spotlight 的键盘优先搜索浮层：居中出现、毛玻璃背景、大圆角、柔和阴影、输入即搜索、方向键选择、回车执行、Esc 关闭。界面只保留搜索和结果所需信息，设置、诊断、安装向导等低频功能通过轻量入口或二级面板承载。

## 可验证结果

- 唤起后窗口位于屏幕水平居中、靠上约 18% 到 22% 的位置，而不是铺满屏幕。
- 默认态只显示单个搜索输入浮层，视觉接近 Spotlight 搜索框。
- 输入后结果列表在搜索框下方自然展开，仍保持同一个浮层整体。
- 全流程可只用键盘完成：输入、上下选择、回车打开、Esc 隐藏、快捷键打开设置。
- 鼠标可用但不抢主流程；按钮数量少，设置入口弱化。
- 亮色和暗色模式都具备毛玻璃、圆角和阴影层次。
- 窗口尺寸、圆角、阴影、间距、选中态都有稳定 token，避免零散样式。

## 当前状态

已有基础：

- `src/renderer/src/styles.css` 中 `.shell` 已使用透明背景、毛玻璃、圆角和阴影。
- `.compactShell` 已支持接近搜索条的紧凑状态。
- `src/renderer/src/App.tsx` 已具备方向键、Enter、Esc、设置快捷键等键盘交互。
- 搜索结果和设置面板已有组件拆分，适合渐进调整。

主要差距：

- 主窗口仍按完整应用壳设计，存在 `workspace`、`contextStrip`、`previewPane`、较多状态信息。
- 结果区像应用面板，不像 Spotlight 的同体下拉列表。
- 视觉色彩偏品牌化绿色，Spotlight 应更克制：系统灰、半透明白/黑、蓝色选中态。
- 设置入口和筛选 pill 过显眼，会削弱“唤起即输入”的焦点。
- 当前 `results` 高度依赖 `100vh`，不适合随结果数量自适应的浮层。

## 设计原则

### 1. 单一焦点

搜索输入是第一视觉层级。唤起后不展示品牌区、侧栏、说明文案或大面积设置内容。用户视线应先落在输入光标，再自然扫到结果列表。

### 2. 浮层不是窗口

主界面应像系统浮层，而不是应用窗口：

- 无可见标题栏。
- 无侧边导航。
- 无卡片嵌套。
- 无大面积装饰背景。
- 内容边界由毛玻璃、圆角和阴影表达。

### 3. 键盘优先

鼠标交互只作为补充。所有核心路径必须能用键盘完成：

- `Alt+Shift+W` 或用户配置快捷键：唤起 QuickTab。
- 输入：即时搜索。
- `ArrowUp` / `ArrowDown`：移动选中项。
- `Enter`：打开选中项或直接搜索/打开 URL。
- `Esc`：清空输入；再次按或空输入时隐藏窗口。
- `Cmd+,` / `Ctrl+,`：打开设置。

### 4. 信息渐进披露

结果行只展示判断打开目标所需的信息：

- favicon 或类型图标。
- 标题。
- 域名或路径。
- 来源类型可弱化为右侧小标签。

预览面板、详细分数、匹配原因、诊断信息不在默认搜索态展示。

## 视觉规格

### 窗口形态

建议由主进程控制窗口尺寸：

- 默认搜索态：宽 `720px`，高 `76px`。
- 有结果态：宽 `720px`，高 `min(560px, 搜索框高度 + 结果高度)`。
- 最小宽度：`560px`。
- 小屏宽度：`calc(100vw - 32px)`。
- 位置：水平居中，垂直位置为屏幕顶部下方约 `18vh`。

如果当前窗口尺寸暂时不改，前端仍应按上述尺寸设计内部浮层，并让背景保持透明。

### 外层浮层

CSS token 建议：

```css
:root {
  --spotlight-radius: 28px;
  --spotlight-search-height: 72px;
  --spotlight-width: 720px;
  --spotlight-blur: 34px;
  --spotlight-border-light: rgba(255, 255, 255, 0.58);
  --spotlight-border-dark: rgba(255, 255, 255, 0.16);
  --spotlight-shadow-light: 0 26px 80px rgba(15, 23, 42, 0.24);
  --spotlight-shadow-dark: 0 30px 90px rgba(0, 0, 0, 0.46);
}
```

浮层样式：

- 圆角：`28px`。
- 背景：亮色 `rgba(248, 250, 252, 0.72)`，暗色 `rgba(30, 32, 36, 0.72)`。
- 模糊：`blur(34px) saturate(1.45)`。
- 边框：1px 半透明高光。
- 阴影：大而软，避免硬边。

### 搜索框

- 高度：`72px`。
- 左右内边距：`24px`。
- 图标：`Search`，尺寸 `25px`，颜色使用系统次级文本色。
- 字号：`26px`，字重 `500`。
- 占位文案：`搜索标签页、书签和历史记录`。
- 输入框无独立边框、无底色，完全融入浮层。
- 清空按钮仅在有输入时出现，使用圆形 `x` 图标，不显示文字。
- 设置按钮放在最右侧，视觉弱化，hover 时才明显。

### 结果列表

- 搜索框与结果列表之间使用 1px 分隔线或极浅内阴影。
- 单行高度：`58px` 到 `64px`。
- 行内布局：`32px 图标 + 标题/URL + 类型标签`。
- 选中态：使用 macOS 风格蓝色或系统强调色，不用绿色大块背景。
- 选中态文字保证对比度，标题白色或接近白色。
- 非选中行 hover 不上浮，只轻微变色。
- 最多展示 8 条，超出滚动。

建议选中态：

```css
.resultItem.selected {
  color: #fff;
  background: linear-gradient(180deg, #2f8cff, #0a73ff);
}
```

暗色模式可保持同一蓝色，但阴影更弱，避免发光感过重。

### 色彩

减少品牌绿的主导占比，保留在连接成功、Safari 权限等状态提示中。主搜索体验使用系统中性色：

- 主文本：亮色 `#1d1d1f`，暗色 `#f5f5f7`。
- 次级文本：亮色 `#6e6e73`，暗色 `#a1a1a6`。
- 分隔线：亮色 `rgba(60, 60, 67, 0.18)`，暗色 `rgba(255, 255, 255, 0.12)`。
- 选中蓝：`#0a84ff` 或 `#007aff`。

### 动效

动效应短、轻、系统感强：

- 唤起：`opacity 0 -> 1`，`translateY(-8px) -> 0`，`scale(0.985) -> 1`，时长 `140ms`。
- 结果展开：高度变化使用 `160ms ease-out`。
- 选中行切换：背景 `90ms`，不做位移。
- 禁止大幅弹跳、旋转、装饰动画。

## 交互规格

### 默认态

无查询、无结果时：

- 只显示搜索框。
- 不显示空状态插图。
- 不显示“连接扩展”大按钮。
- 可在搜索框底部极弱展示一行快捷键提示，但建议只在首次使用或 onboarding 中展示。

### 输入态

有查询时：

- 结果列表立即展开。
- 第一条结果自动选中。
- 如果输入是 URL，第一条可为“打开该网址”。
- 如果无结果，显示一行紧凑空状态：`没有匹配结果`，不要使用大面积空态。

### 执行态

- `Enter` 打开当前选中项。
- 执行后立即隐藏窗口，避免用户感觉应用停留。
- 执行失败时再展开错误行或 toast，避免默认占空间。

### 设置态

设置不应破坏 Spotlight 主体验。建议改成两种方案之一：

方案 A：同窗口二级 sheet

- `Cmd+,` 打开设置 sheet。
- sheet 覆盖在当前浮层上，宽度不超过 `720px`，最大高度 `640px`。
- 背景保持毛玻璃，但内容可以更密集。

方案 B：独立设置窗口

- 主搜索浮层只负责搜索。
- 设置入口打开独立窗口，适合复杂配置、诊断、卸载等操作。
- 这更接近 Spotlight 的轻量主入口，也降低主浮层复杂度。

推荐先做方案 A，改动较小。

## 结构调整建议

### React 结构

当前 `App.tsx` 可先保持单文件，避免重构过大。建议只做命名和条件渲染调整：

- `.shell` 作为透明窗口容器。
- `.appChrome` 改为 Spotlight 浮层主体。
- `.workspace` 保留，但变成居中布局容器。
- `.assistantBar` 和 `.searchBox` 合并为主搜索区域视觉。
- `.contextStrip` 默认隐藏，筛选模式改为键盘或设置中的配置。
- `.previewPane` 默认移除或仅在设置中开启实验模式。

后续再按需要拆分：

- `SpotlightSearchBox`
- `SpotlightResults`
- `SettingsSheet`
- `OnboardingSheet`

### CSS 调整重点

优先修改：

- `.shell`
- `.appChrome`
- `.workspace`
- `.searchBox`
- `.contextStrip`
- `.searchStage`
- `.resultColumn`
- `.results`
- `.resultItem`
- `.settingsSheet`

避免优先动：

- Safari 权限逻辑。
- 搜索排序逻辑。
- 浏览器桥接逻辑。
- 快捷键注册逻辑。

## 分阶段实施计划

### 阶段 1：主浮层外观

改动：

- 定义 Spotlight 视觉 token。
- 调整 `.shell` 和 `.workspace`，让内容居中并限制宽度。
- 将默认搜索态压缩为单个大圆角搜索浮层。

验证：

- 启动应用后只看到居中搜索浮层。
- 亮色/暗色模式下边框、阴影和毛玻璃都可见。
- 文本没有溢出，窗口无滚动条。

### 阶段 2：结果列表 Spotlight 化

改动：

- 结果列表与搜索框同属一个浮层。
- 移除结果面板卡片感。
- 改选中态为系统蓝。
- 限制最多可视 8 条结果。

验证：

- 输入查询后结果自然展开。
- 上下键移动选中项无布局跳动。
- 长标题、长 URL 单行省略。
- 无结果时只显示紧凑提示。

### 阶段 3：弱化低频功能

改动：

- 默认隐藏 `contextStrip`。
- 设置按钮弱化，只保留图标。
- 设置内容作为 sheet 展示。
- onboarding 和诊断保持二级视图，不进入默认搜索态。

验证：

- 用户唤起后不会先读到设置、状态、筛选等信息。
- `Cmd+,` / `Ctrl+,` 可打开设置。
- 设置关闭后焦点回到搜索框。

### 阶段 4：键盘和窗口体验收口

改动：

- 确认 Esc 行为：有输入先清空，无输入隐藏。
- 执行结果后隐藏窗口。
- 主进程按搜索态/结果态调整窗口高度。
- 窗口显示位置固定为屏幕上方居中。

验证：

- 全流程不使用鼠标可完成。
- 窗口唤起位置稳定。
- 多屏场景下出现在当前焦点屏幕。
- 打开结果后窗口消失。

## 验收清单

- [ ] 默认态只显示一个居中搜索浮层。
- [ ] 搜索框高度、圆角、字号接近 Spotlight。
- [ ] 背景透明，浮层具备毛玻璃和柔和阴影。
- [ ] 输入后结果列表与搜索框连成一个整体。
- [ ] 第一条结果自动选中。
- [ ] 方向键选择、Enter 执行、Esc 隐藏可用。
- [ ] 设置入口存在但不喧宾夺主。
- [ ] 无结果状态紧凑，不出现大面积空态。
- [ ] 亮色和暗色模式都可读。
- [ ] 窗口尺寸不会因长文本抖动。
- [ ] 不影响 Safari、Chrome、Edge 数据同步逻辑。
- [ ] `npm test`、`npm run typecheck`、`npm run build` 通过。

## 建议优先级

P0：

- 主浮层居中。
- 搜索框 Spotlight 化。
- 结果列表与搜索框合并。
- 键盘路径完整。

P1：

- 设置 sheet 化。
- 主进程动态窗口高度。
- 亮暗主题精修。

P2：

- 多屏定位优化。
- 首次使用轻提示。
- 可选的 favicon 支持。

## 不建议做的事

- 不要加入品牌宣传区、欢迎页或大标题。
- 不要把设置、诊断、安装向导放在默认搜索视图里。
- 不要使用强烈渐变、装饰光斑或复杂背景图。
- 不要让按钮 hover 产生上浮阴影，Spotlight 风格应更安静。
- 不要为了风格重写搜索、桥接或排序逻辑。

## 推荐下一步

先实现阶段 1 和阶段 2。它们直接决定用户对“像不像 Spotlight”的第一印象，且主要集中在 `src/renderer/src/styles.css` 和少量 `App.tsx` 条件渲染，风险可控、验证明确。
